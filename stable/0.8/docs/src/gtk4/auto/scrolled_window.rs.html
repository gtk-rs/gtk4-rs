    /// [`ScrolledWindow`][crate::ScrolledWindow] is a container that makes its child scrollable.
    ///
    /// It does so using either internally added scrollbars or externally
    /// associated adjustments, and optionally draws a frame around the child.
    ///
    /// Widgets with native scrolling support, i.e. those whose classes implement
    /// the [`Scrollable`][crate::Scrollable] interface, are added directly. For other types
    /// of widget, the class [`Viewport`][crate::Viewport] acts as an adaptor, giving
    /// scrollability to other widgets. [`set_child()`][Self::set_child()]
    /// intelligently accounts for whether or not the added child is a [`Scrollable`][crate::Scrollable].
    /// If it isn’t, then it wraps the child in a [`Viewport`][crate::Viewport]. Therefore, you can
    /// just add any child widget and not worry about the details.
    ///
    /// If [`set_child()`][Self::set_child()] has added a [`Viewport`][crate::Viewport] for you,
    /// it will be automatically removed when you unset the child.
    /// Unless [`hscrollbar-policy`][struct@crate::ScrolledWindow#hscrollbar-policy] and
    /// [`vscrollbar-policy`][struct@crate::ScrolledWindow#vscrollbar-policy] are [`PolicyType::Never`][crate::PolicyType::Never] or
    /// [`PolicyType::External`][crate::PolicyType::External], [`ScrolledWindow`][crate::ScrolledWindow] adds internal [`Scrollbar`][crate::Scrollbar] widgets
    /// around its child. The scroll position of the child, and if applicable the
    /// scrollbars, is controlled by the [`hadjustment`][struct@crate::ScrolledWindow#hadjustment]
    /// and [`vadjustment`][struct@crate::ScrolledWindow#vadjustment] that are associated with the
    /// [`ScrolledWindow`][crate::ScrolledWindow]. See the docs on [`Scrollbar`][crate::Scrollbar] for the details,
    /// but note that the “step_increment” and “page_increment” fields are only
    /// effective if the policy causes scrollbars to be present.
    ///
